# 제품 요구사항 정의서 (PRD): LLM 성능 테스트 솔루션

## 1. 개요 (Product Overview)
본 문서는 내부 서버에서 동작 중인 LLM(Large Language Model) 서비스의 토큰 생성 속도 및 안정성을 측정하기 위한 성능 테스트 프로그램의 요구사항을 정의합니다. 

## 2. 주요 목표 (Key Objectives)
- 사용자 친화적인 UI를 통해 비전문가도 쉽게 LLM 성능 및 부하 테스트 수행
- 실사용 환경과 유사한 동시 접속자 부하 현상 시뮬레이션
- 주요 로컬 서빙 플랫폼들과의 유연한 연동
- 생성형 AI에 특화된 핵심 성능 지표(TTFT, TPOT, Latency 등) 도출 및 시각화

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 테스트 대상 서버 설정
- **주소 입력란**: 사용자로부터 대상 서버의 엔드포인트(URL/IP 및 포트)를 직접 입력받는 기능
- **플랫폼 호환성**: 다음의 주요 로컬 서빙 플랫폼들을 지원 (주로 OpenAI 호환 API 활용)
  - vLLM
  - LiteLLM
- **모델별 특화 파라미터 제어**: 테스트 대상 모델의 특성에 맞춘 세부 생성 파라미터를 유연하게 설정할 수 있는 기능 제공
  - 예시: gpt-oss 계열 등 추론 모델의 `reasoning_effrot` 조절 기능
  - 예시: Qwen3 모델의 `enable_thinking` 활성화 여부(True/False) 옵션 부여 등 모델별 커스텀 API 파라미터 전달 지원

### 3.2. 성능 지표 측정 (Metrics)
다음과 같은 LLM 특화 퍼포먼스 지표를 필수적으로 측정하고 결과값으로 제공해야 합니다:
- **TTFT (Time To First Token)**: 사용자 요청 후 첫 번째 응답 토큰이 생성될 때까지의 대기 시간
- **TPOT (Time Per Output Token)**: 첫 토큰 생성 후, 이후 각 토큰이 생성되는 데 걸리는 평균 시간
- **FPS/TPS (Frames/Tokens Per Second)**: 초당 생성되는 평균 출력 토큰 수 (토큰 생성 대역폭)
- **Latency (전체 지연 시간)**: 단일 요청에 대한 전체 응답이 완료되기까지 소요된 총 시간
- **Token Usage (토큰 사용량)**: 요청 처리에 소비된 전체 토큰 사용량으로, 입력(Input/Prompt) 토큰과 출력(Output/Completion) 토큰을 각각 명시적으로 구분하여 제공

### 3.3. 부하 테스트 (Load Testing)
- **동시 접속자(Concurrency) 설정**: 시스템 스트레스 및 부하 테스트를 위해 가상의 동시 접속 유저 수를 지정하는 기능
- **테스트 파라미터 제어**: 부하를 발생시킬 입력 프롬프트(텍스트), 총 테스트 횟수 또는 테스트 유지 지속 시간 설정 기능
- **안정성 모니터링**: 부하 증가 시 발생하는 타임아웃, 응답 성공률 및 에러율(Error Rate) 추적

### 3.4. 사용자 인터페이스 (UI/UX)
- **직관적인 조작**: 터미널 명령어나 코딩 지식이 없는 비전문가도 쉽게 클릭만으로 테스트를 구성하고 실행할 수 있는 GUI(웹 기반 또는 데스크톱 애플리케이션) 제공
- **결과 시각화(대시보드)**: 측정된 지표들을 차트, 그래프 등으로 시각화하여 현재 시스템의 병목 현상이나 성능을 한눈에 파악할 수 있도록 구성
- **결과 초기화**: 누적된 테스트 결과와 차트를 UI 상에서 즉시 초기화할 수 있는 Clear 버튼 제공
- **리포트 내보내기**: 테스트 결과를 문서화하기 위해, 테스트 설정값(Configuration)과 실행 일시가 포함된 JSON 원본 데이터 파일 및 마크다운(MD) 리포트 다운로드 기능 제공 (파일명에 타임스탬프 적용)

## 4. 비기능 요구사항 (Non-Functional Requirements)
- **경량성**: 프로그램 자체가 시스템 리소스를 과도하게 점유하여 대상 서버의 성능 측정 결과에 영향을 주지 않아야 함
- **확장성**: 추후 새로운 서빙 플랫폼의 추가 연동이나 새로운 측정 지표(예: Prompt Processing Time) 도입이 용이하도록 모듈화된 아키텍처 채택

## 5. 제외 사항 (Out of Scope)
- 모델에서 생성된 결과물의 텍스트 품질(Quality)이나 정확도(Accuracy) 평가는 본 테스트 프로그램의 범위에서 제외하며, 오직 **모델의 속도 및 서버의 처리 능력(Throughput)** 측정에 집중함.
