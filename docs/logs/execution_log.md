# LLM 성능 테스트 솔루션 개발 실행 로그

## 진행 일자: 2026년 3월 15일 ~ 16일

### 주요 실행 내역
1. **프로젝트 초기화 및 기본 구조 작성**
   - 사용자 요구사항에 따라 `FastAPI` + `Vanilla JS` 조합으로 솔루션 아키텍처 확정.
   - 실행 구성을 단순화하기 위해 진입점 스크립트인 `run.sh` 작성.

2. **백엔드 로직 구현 완료**
   - 비동기 LLM 클라이언트 클래스 구성 완료.
   - 워커 태스크가 부하를 발생시키고 SSE(Server-Sent Events)를 통해 결과를 프론트엔드로 브로드캐스팅하는 흐름 완성.
   - 상대경로 Import 에러 수정 (모듈 인식 범위 문제 해결).

3. **기본 UI 및 차트 기능 연동**
   - Glassmorphism 테마의 대시보드 구조 완성.
   - `Chart.js`를 이용한 Latency/TPS 지표 시각화 및 테이블 로깅 구현.

4. **사용자 피드백 및 기능 고도화**
   - Chart.js 반복 테스트 동작 시 캔버스 충돌 에러(`Canvas already in error state`) 해결 (상세 원인은 errors 디렉토리 참고).
   - 무한정 길어지는 차트 레이아웃(Y축 팽창 현상) CSS Fix 처리.
   - Duration(지속 시간) 기반 테스트 종료 기능 추가.
   - CSV Export 기능을 **JSON 원본 추출** 및 **Markdown Report 다운로드** 기능으로 고도화 완료.
   - 다운로드 파일명에 동적으로 타임스탬프(`YYYYMMDD_HHMM`) 추가 처리.
   - 결과(Results) 테이블 및 차트 정보를 손쉽게 지울 수 있는 `Clear Results` 버튼 구현 완료.
